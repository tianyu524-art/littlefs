# PWR Autotest

This directory contains an automated stress test for the `/lfs0/LOG/PWR` path.

## Files

- `pwr_autotest.c`
  - Main test program
- `build_autotest.ps1`
  - Builds `pwr_autotest.exe`
- `run_autotest.ps1`
  - Runs the test
- `autotest.txt`
  - Execution log generated by the test

## Safety

The test does **not** modify the original `simulator\test001.img` in place.

It first copies:

- source image: `simulator\test001.img`

to:

- work image: `autotest\test001_work.img`

and runs all operations on the work image.

## Behavior

For each iteration, the program:

1. Mounts the work image
2. Uses target directory `/lfs0/LOG/PWR`
3. Ensures `PWR.IDX` exists
4. Creates 1 numbered `.PWR` file
5. Writes random data with size chosen from `12244`, `28204`, `30604`
6. Writes the new file name and length into `PWR.IDX`
7. If the current number of `.PWR` files is greater than 5, deletes the smallest numbered `.PWR` file
9. Writes the current `.PWR` listing into `PWR.IDX.tmp`
10. Renames `PWR.IDX.tmp` to `PWR.IDX`
11. Runs an internal `lschk`-style scan
12. Stops if any entry is not `realfile`, or after the iteration limit

## Notes

- The file name format used is `00000000.PWR`
- This format is 12 characters long including the dot
- The original requirement mentioned 11 characters, but the explicit format string was used

## Usage

Build:

```powershell
.\build_autotest.ps1
```

Run with default 1000 iterations:

```powershell
.\run_autotest.ps1
```

Run with a smaller iteration count:

```powershell
.\run_autotest.ps1 10
```
